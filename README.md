# Pixel Social Chat

Тестовый README проекта.

## Идея

**Pixel Social Chat** — это социальный чат с интерактивным пиксельным фоном.  
Вместо обычного окна переписки у вас есть общая комната, где находятся аватары двух собеседников, и ими можно управлять через объекты окружения.

Пример: нажимаешь на кровать — персонаж идет спать.

## Что делает продукт особенным

- **Чат + мини-игра в одном экране**: переписка и живое взаимодействие в комнате.
- **Два аватара в реальном времени**: у каждого участника свой персонаж.
- **Интерактивные предметы**: кровать, стул, стол и другие объекты с действиями.
- **Совместное обустройство**: комнату можно заполнять мебелью вместе.
- **Контент-модель**: часть мебели бесплатная, часть — платная (косметика/премиум).

## MVP (первая версия)

Планируемый набор для старта:

1. Локальный мессенджер (без удаленных серверов).
2. Одна базовая комната.
3. По одному аватару на пользователя.
4. Набор простых интеракций с мебелью.
5. Бесплатный каталог мебели + несколько платных предметов.

## Технические вводные

- На старте проект **локальный**, поэтому можно обойтись без сложной инфраструктуры.
- Полноценное шифрование и дорогая серверная часть для MVP не обязательны.
- Пиксельный стиль снижает требования к производительности и упрощает визуальную часть.

## Что лучше для старта: сайт или приложение?

### Рекомендация: начать с веб-версии (сайта)

Почему:

- **Быстрый запуск**: не нужно публиковать в сторах.
- **Ниже порог входа**: пользователь открывает ссылку и сразу пробует.
- **Дешевле проверять гипотезы**: проще выпускать обновления и собирать фидбек.
- **Пиксельный стиль хорошо работает в браузере**.

### Когда переходить к приложению

Имеет смысл делать отдельное приложение (или упаковать веб в desktop/mobile), когда:

- нужна более глубокая интеграция с ОС;
- появятся офлайн-сценарии;
- потребуется продвинутая производительность и фоновая работа.

## Возможный стек для MVP

- Frontend: `React` + `TypeScript`
- Рендер комнаты: `Phaser` или `PixiJS`
- Локальное хранение: `IndexedDB` / `localStorage`
- Desktop-оболочка (опционально): `Tauri` или `Electron`

## Ближайшие шаги

1. Утвердить платформу MVP (рекомендуется web).
2. Описать базовые действия аватара и предметов.
3. Собрать прототип одной комнаты.
4. Добавить локальный чат и синхронизацию состояния в рамках сессии.
5. Провести первые тесты с пользователями.

## Быстрый старт разработки

### Frontend

```bash
cd frontend
npm install
npm run dev
```

### Backend

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload
```
